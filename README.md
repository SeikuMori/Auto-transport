# Django Person Card — Полное описание проекта

Это приложение на Django предоставляет простую систему CRUD для хранения "карточек" людей (Person). Проект ориентирован на локальное использование и демонстрацию практик Django: модели, админка, представления на основе классов, шаблоны, загрузка файлов, простая локализация через properties-файлы и собственная middleware для переключения языка.

Обзор
------
Приложение хранит записи о людях (Фамилия, Имя, Отчество, Дата рождения, Фото). Предоставлены веб-интерфейсы для:
- просмотра списка людей (с поиском и пагинацией),
- добавления новой записи,
- редактирования существующей,
- просмотра детальной информации,
- удаления записи.

Функциональность
-----------------
- CRUD для сущности `Person` (создание, чтение, обновление, удаление).
- Поиск по фамилии/имени (параметр GET `q` в списке).
- Пагинация списка (`paginate_by=10`).
- Загрузка фотографии: файлы сохраняются в `MEDIA_ROOT/persons/`.
- Простой механизм локализации через properties-файлы (`cards/locale/<lang>/messages.properties`).
- Кастомная middleware `LocaleMiddleware` для переключения языка через GET-параметр `lang`.
- Контекст-процессор `cards.context_processors.translations`, который добавляет словарь переводов в шаблоны как `t`.

Требования
-----------
- Python 3.10+ (рекомендуется современная версия 3.10+)
- Django 4.x/5.x (в проекте используется настройка, совместимая с Django 5.x)
- PostgreSQL (в `settings.py` по умолчанию настроен engine `django.db.backends.postgresql`)
- Дополнительные зависимости см. `requirements.txt` (например, `jproperties`, `django_bootstrap5`).

Установка и запуск (локально)
---------------------------
1. Клонируйте репозиторий и перейдите в каталог проекта:

```powershell
git clone <repository-url>
cd django-person-card
```

2. Создайте виртуальное окружение и активируйте его:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # PowerShell
```

3. Установите зависимости:

```powershell
pip install -r requirements.txt
```

4. Настройте параметры подключения к базе в `personal_card_system/settings.py` (или используйте переменные окружения/другой способ конфигурации). По умолчанию там указаны:
- NAME: `personal_card_db`
- USER: `postgres`
- PASSWORD: `12345`
- HOST: `localhost`
- PORT: `5432`

5. Выполните миграции и создайте суперпользователя для доступа в админку:

```powershell
python manage.py migrate
python manage.py createsuperuser
```

6. Запустите сервер разработки:

```powershell
python manage.py runserver
```

7. Откройте в браузере: `http://127.0.0.1:8000/` — корень приложения содержит список карточек (приложение подключено через `personal_card_system/urls.py`).

Структура проекта (важные файлы и папки)
-------------------------------------
- `cards/` — приложение Django, реализующее функциональность карточек:
  - `models.py` — модель `Person` (поля: `last_name`, `first_name`, `middle_name`, `birth_date`, `photo`).
  - `views.py` — представления на основе классов: `PersonListView`, `PersonCreateView`, `PersonDetailView`, `PersonUpdateView`, `PersonDeleteView`.
  - `urls.py` — маршруты приложения (имя пространства — `cards`).
  - `admin.py` — регистрация модели в админке и отображение превью фото.
  - `translation.py` — загрузка `.properties`-файлов через `jproperties` и преобразование в вложенный словарь для шаблонов.
  - `context_processors.py` — добавляет `t` в контекст шаблонов (переводы).
  - `middleware.py` — `LocaleMiddleware` для управления языком через GET-параметр `lang` и сессию.
  - `locale/<lang>/messages.properties` — файлы переводов для каждого языка (напр., `ru`, `en`).
- `templates/` — директория шаблонов проекта:
  - `templates/base.html` — базовый шаблон с подключением Bootstrap и общими частями UI.
  - `templates/cards/*.html` — шаблоны списка, формы, детальной страницы и подтверждения удаления.
- `static/` — статические ресурсы (CSS, JS).
- `media/` — загружаемые файлы (фото людей).
- `personal_card_system/settings.py` — основные настройки Django (БД, `MEDIA_ROOT`, `STATICFILES_DIRS`, `INSTALLED_APPS`, `MIDDLEWARE`, и т.д.).

Архитектура и ключевые компоненты
--------------------------------
1. Модель `Person` (`cards/models.py`)
   - Поля: `last_name` (CharField), `first_name` (CharField), `middle_name` (CharField, optional), `birth_date` (DateField), `photo` (ImageField).
   - Валидация: для текстовых полей применяются RegexValidator'ы, ожидающие кириллические буквы (в проекте рассчитано на русскоязычный ввод).

2. Представления (CBV) (`cards/views.py`)
   - `PersonListView`: выводит список, реализует поиск по GET-параметру `q` (фильтрация по фамилии и имени), пагинацию.
   - `PersonCreateView` / `PersonUpdateView`: формы создания/редактирования; переопределён `get_form`, чтобы подставлять локализованные метки и placeholder'ы из переводов.
   - `PersonDetailView`: показывает одну запись.
   - `PersonDeleteView`: подтверждение и удаление записи (через POST).

3. URLs
   - `''` → список (`cards:person_list`)
   - `add/` → создание (`cards:person_add`)
   - `<int:pk>/` → детально (`cards:person_detail`)
   - `<int:pk>/edit/` → редактирование (`cards:person_edit`)
   - `<int:pk>/delete/` → удаление (`cards:person_delete`)

4. Контекст-процессор и перевод
   - `cards/context_processors.translations` читает текущий язык из сессии и добавляет в контекст шаблонов переменную `t` — вложенный словарь с переводами.
   - `cards/translation.get_translation(lang)` загружает `cards/locale/<lang>/messages.properties` с помощью `jproperties`, преобразует ключи с точками в вложенные словари (например, `label.last_name` → `t['label']['last_name']`).

5. Middleware для локали
   - `cards.middleware.LocaleMiddleware` смотрит GET-параметр `lang` и, если установлен, сохраняет значение в сессии (`request.session['language']`) и помещает текущее значение в `request.LANGUAGE`.
   - Это лёгкий способ переключения языка без использования полного встроенного механизма i18n Django.

Валидация и формы
------------------
- Валидация текста реализована через `RegexValidator`, который проверяет кириллицу (в `models.py`).
- Поля формы в Create/Update меняют свои `label` и `placeholder` в `get_form`, используя данные из `get_translation`.

Шаблоны, static и media
------------------------
- Шаблоны используют `django_bootstrap5` для стилизации форм и Bootstrap-классы для разметки.
- Статические файлы находятся в `static/` и подключаются через `STATICFILES_DIRS`.
- Пользовательские загружаемые изображения сохраняются в `MEDIA_ROOT/persons/` и доступны по `MEDIA_URL`.

База данных и миграции
----------------------
- По умолчанию в `settings.py` настроен PostgreSQL; для локальной отладки можно переключиться на SQLite (заменить `DATABASES['default']`).
- Миграции находятся в `cards/migrations/`; создавать/применять миграции нужно стандартными командами Django `makemigrations`/`migrate`.

Тестирование
-----------
- Тесты приложения (если добавлены) находятся в `cards/tests.py`.
- Запуск тестов: `python manage.py test`.

Безопасность и рекомендации по деплою
-----------------------------------
- В `settings.py` стоит `DEBUG = False` — перед деплоем убедитесь, что `ALLOWED_HOSTS` настроены правильно и секретный ключ защищён.
- Для продакшна настройте статические файлы и `MEDIA` (например, используйте S3 или отдельный файловый сервер) и используйте `collectstatic`.
- Логи: внутри middleware используются `print()` для отладки; в продакшне замените их на вызовы логгера (`logging`).

Полезные сценарии использования
-------------------------------
- Переключение языка в браузере: добавьте к URL `?lang=en` или `?lang=ru` — middleware сохранит выбор в сессии.
- Поиск: на странице списка добавьте `?q=Иванов` для поиска по фамилии/имени.
