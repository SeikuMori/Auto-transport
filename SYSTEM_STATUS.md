## 🚀 ГОТОВО К ИСПОЛЬЗОВАНИЮ

### 1. Django Admin Interface (ПОЛНОСТЬЮ РАБОЧАЯ)
**URL:** `http://127.0.0.1:8000/admin/`

#### Доступные модели:
- **Профиль пользователя** (UserProfile)
  - ФИО, табельный номер, должность (4 роли)
  - Связь с Django User
  - Синхронизация с системой разрешений

- **Справочники** (Reference Models)
  - Марки и модели автомобилей
  - Цвета кузова
  - Типы кузова
  - Категории ТС
  - Типы топлива
  - Детальные типы кузова

- **Транспортные средства** (Vehicle) - 100+ ПОЛЕЙ в 4 БЛОКАХ
  - **Блок 1: Паспорт ТС**
    - Гаражный номер, VIN
    - Марка, модель, цвет
    - ПТС (бумажный): серия, номер, дата, скан
    - ЭПТС: номер, дата, скан
    - Фотография ТС

  - **Блок 2: Регистрационные данные ГИБДД**
    - ГРЗ (государственный регистрационный номер)
    - Свидетельство регистрации
    - ОСАГО: номер, даты выдачи/истечения, стоимость, скан
    - Диагностическая карта: номер, даты, скан

  - **Блок 3: Общие характеристики**
    - Технические характеристики (масса, объем двигателя, мощность)
    - Номера шасси и кузова, год выпуска
    - Пробег (км) и моточасы (ч)
    - Грузоподъемность, вместимость пассажиров
    - Ёмкость топливного бака
    - ГБО: наличие, тип, объём, даты проверок, скан
    - Тахограф: наличие, даты калибровки
    - ДПОГ: даты проверок, скан
    - Статус ТС, период ТО-2

  - **Блок 4: Документы по срокам и Бухгалтерские данные**
    - Сроки техосмотра и страховки
    - Периоды ТО-2 и ТО-3 (дней)
    - Код ОКОФ, дата ввода в эксплуатацию
    - Амортизация (ставка, накопленная, остаточная стоимость)

  - **Архивирование**
    - Статус архивирования
    - Дата и пользователь архивирования

- **Техническое обслуживание (ТО-2)** (MaintenanceSchedule)
  - История обслуживания (дата, пробег)
  - Следующее обслуживание (дата, пробег)
  - Статус (Запланировано, В процессе, Завершено, Просрочено, Отменено)
  - Заметки и результаты
  - **Встроено в форму ТС** как inline редактирование

- **Журнал аудита** (AuditLog) - ТОЛЬКО ДЛЯ ЧТЕНИЯ
  - ФИО пользователя, логин
  - Тип действия (CREATE, UPDATE, DELETE, VIEW)
  - Описание объекта
  - Внесённые изменения
  - Дата/время и IP адрес

#### Возможности в админке:
✓ Полный CRUD для всех моделей (кроме AuditLog - только чтение)
✓ Фильтрация по 10+ критериям (архив, статус, тип, цвет и т.д.)
✓ Поиск по гаражному номеру, VIN коду, марке
✓ Превью фотографий ТС
✓ Превью/скачивание сканов документов
✓ Встроенное редактирование расписания ТО-2
✓ Цветное отображение статусов
✓ Защита с LoginRequired

### 2. Система ролевого доступа (ПОЛНОСТЬЮ РЕАЛИЗОВАНА)

**Четыре роли с разными правами:**
```
Администратор (Admin)         → 12 прав (полный доступ)
Руководитель (Manager)        → 10 прав (полный CRUD)
Специалист (Specialist)       → 8 прав (добавление и редактирование)
Пользователь (Viewer)         → 4 прав (только просмотр)
```

**Инициализация прав:**
```bash
python manage.py setup_roles          # Создать группы и права
python manage.py sync_user_roles      # Синхронизировать пользователей с группами
python manage.py sync_user_roles --user <username>  # Синхронизировать одного пользователя
```

**После инициализации:**
- Отправить пользователей в соответствующие группы через админку
- Система будет проверять права при доступе к представлениям

## 📊 Модели Данных

**Отношения между моделями:**
```
UserProfile (OneToOne) → User
      ↓
   position = {admin, supervisor, specialist, user}

Vehicle (Main Model)
├─ ForeignKey → BrandModel
├─ ForeignKey → Color
├─ ForeignKey → VehicleType
├─ ForeignKey → Category
├─ ForeignKey → FuelType
├─ ForeignKey → BodyType
├─ OneToOne → User (archived_by)
└─ Reverse FK ← MaintenanceSchedule

MaintenanceSchedule
├─ ForeignKey → Vehicle
└─ status = {scheduled, in_progress, completed, overdue, cancelled}

AuditLog
├─ Отслеживает все действия с Vehicle
└─ action = {CREATE, UPDATE, DELETE, VIEW}
```
